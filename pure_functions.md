# Модуль 2: Чистые функции (Pure Functions)

Чистая функция — это фундаментальный строительный блок функционального программирования. 

## 1. Определение
Функция считается чистой, если она удовлетворяет двум условиям:
1. **Детерминированность**: Для одних и тех же входных данных она всегда возвращает один и тот же результат.
2. **Отсутствие побочных эффектов (Side Effects)**: Она не изменяет ничего за пределами своей области видимости и не взаимодействует с "внешним миром" (не меняет глобальные переменные, не пишет в консоль, не делает сетевых запросов).

## 2. Примеры

### Грязная функция (Impure)
```javascript
let tax = 0.2;

const calculateTotal = (price) => {
  return price + (price * tax); // Зависит от внешней переменной tax
};

console.log(calculateTotal(100)); // 120
tax = 0.3;
console.log(calculateTotal(100)); // 130 (результат изменился при тех же входных данных)
```

### Чистая функция (Pure)
```javascript
const calculateTotal = (price, taxRate) => {
  return price + (price * taxRate);
};

console.log(calculateTotal(100, 0.2)); // 120
console.log(calculateTotal(100, 0.2)); // 120 (всегда один результат)
```

## 3. Ссылочная прозрачность (Referential Transparency)
Это свойство программы, позволяющее заменить вызов функции её результатом без изменения смысла программы. 

Если мы знаем, что `square(4)` всегда равно `16`, мы можем заменить все вызовы `square(4)` на число `16`. Это позволяет компиляторам и разработчикам легко оптимизировать и рассуждать о коде.

## 4. Побочные эффекты
Что именно считается побочным эффектом?
- Изменение глобальной переменной или свойства объекта.
- Запись в консоль (`console.log`).
- Запись в файл или базу данных.
- Сетевые запросы (`fetch`).
- Манипуляции с DOM.
- Генерация случайных чисел (`Math.random()`) — нарушает детерминизм.
- Получение текущего времени (`Date.now()`) — нарушает детерминизм.

## 5. Преимущества
1. **Тестируемость**: Не нужно настраивать сложное окружение (mocks), достаточно передать аргументы.
2. **Предсказуемость**: Легко понять, что делает функция, просто глядя на её сигнатуру.
3. **Кэширование (Мемоизация)**: Поскольку результат зависит только от аргументов, мы можем сохранять результаты тяжелых вычислений.
4. **Параллелизм**: Чистые функции можно выполнять параллельно, так как они не делят общее состояние.
