# Модуль 1: Введение в функциональное программирование

Функциональное программирование (ФП) — это парадигма программирования, в которой процесс вычисления трактуется как вычисление значений функций в математическом понимании. В отличие от императивного программирования, ФП избегает изменения состояния и мутабельных данных.

## 1. Императивное vs Декларативное

### Императивный подход
Описывает **КАК** компьютер должен выполнить задачу. Он полагается на последовательность команд, циклы и изменение переменных.

**Пример (найти сумму четных чисел в массиве):**
```javascript
const numbers = [1, 2, 3, 4, 5, 6];
let sum = 0;

for (let i = 0; i < numbers.length; i++) {
  if (numbers[i] % 2 === 0) {
    sum += numbers[i];
  }
}

console.log(sum); // 12
```
*Пошаговое объяснение:*
1. Создаем переменную `sum` для хранения промежуточного состояния.
2. Запускаем цикл `for`, управляя индексом `i`.
3. На каждой итерации проверяем условие.
4. Если условие верно, мутируем (изменяем) переменную `sum`.

### Декларативный подход
Описывает **ЧТО** мы хотим получить. Мы используем абстракции (функции), которые скрывают детали реализации циклов.

**Пример (тот же результат в стиле ФП):**
```javascript
const numbers = [1, 2, 3, 4, 5, 6];

const isEven = n => n % 2 === 0;
const sum = (acc, val) => acc + val;

const result = numbers
  .filter(isEven)
  .reduce(sum, 0);

console.log(result); // 12
```
*Пошаговое объяснение:*
1. Определяем правило фильтрации (`isEven`).
2. Определяем правило сложения (`sum`).
3. Применяем `filter` для отбора четных чисел.
4. Применяем `reduce` для вычисления итоговой суммы.
5. Мы не управляем циклами и не меняем переменные напрямую.

## 2. Столпы ФП

1. **Функции — граждане первого класса (First-Class Citizens)**:
   Функции можно сохранять в переменные, передавать как аргументы и возвращать из других функций. Это основа для создания гибких абстракций.
   
2. **Чистота (Purity)**:
   Функции зависят только от своих входных данных и не производят побочных эффектов. Это делает код предсказуемым и легким для тестирования.

3. **Иммутабельность (Immutability)**:
   Вместо изменения существующих данных мы всегда создаем новые. Это исключает целый класс багов, связанных с неожиданным изменением состояния в разных частях программы.
