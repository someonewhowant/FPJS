# Модуль 8: Рекурсия (Recursion)

Рекурсия — это техника, при которой функция вызывает саму себя для решения подзадач меньшего размера. В ФП рекурсия заменяет циклы, так как циклы обычно требуют мутации переменной-счетчика.

## 1. Анатомия рекурсии
Любая рекурсивная функция должна иметь две части:
1. **Базовый случай (Base Case)**: Условие выхода, при котором функция перестает вызывать себя.
2. **Рекурсивный шаг (Recursive Step)**: Вызов функцией самой себя с новыми аргументами, приближающими нас к базовому случаю.

**Пример (факториал):**
```javascript
const factorial = (n) => {
  if (n === 0) return 1; // Базовый случай
  return n * factorial(n - 1); // Рекурсивный шаг
};

console.log(factorial(5)); // 120
```

## 2. Хвостовая рекурсия (Tail Call Optimization - TCO)
Обычная рекурсия может привести к переполнению стека (Stack Overflow) при большом количестве вызовов. Хвостовая рекурсия — это когда рекурсивный вызов является *последним* действием в функции.

**Оптимизированный факториал:**
```javascript
const factorial = (n, acc = 1) => {
  if (n === 0) return acc;
  return factorial(n - 1, n * acc); // Результат вызова сразу возвращается
};
```
Некоторые современные движки JS могут оптимизировать такие вызовы, чтобы не раздувать стек.

## 3. Рекурсия и структуры данных
Рекурсия идеально подходит для работы с древовидными структурами.

**Пример (сумма всех чисел в глубоко вложенном массиве):**
```javascript
const sumDeep = (arr) => {
  return arr.reduce((acc, item) => {
    if (Array.isArray(item)) {
      return acc + sumDeep(item); // Рекурсия для вложенного массива
    }
    return acc + item;
  }, 0);
};

const data = [1, [2, [3, 4], 5], 6];
console.log(sumDeep(data)); // 21
```

## 4. Зачем это в ФП?
- **Иммутабельность**: Нам не нужны `let i = 0` и `i++`.
- **Элегантность**: Многие алгоритмы (сортировка, обход деревьев) описываются рекурсивно гораздо проще, чем итеративно.
- **Декларативность**: Мы описываем "что такое результат", а не "как его высчитать по шагам".
