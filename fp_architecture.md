# Модуль 10: Функциональная архитектура (ACD)

В функциональном программировании мы делим весь код на три фундаментальные категории: **Actions** (Действия), **Calculations** (Вычисления) и **Data** (Данные). Этот подход помогает изолировать побочные эффекты и сделать систему надежной.

## 1. Data (Данные)
Это инертные факты. Они не выполняются, не имеют логики и просто описывают состояние.
- **Примеры**: JSON-объекты, строки, числа, массивы настроек.
- **Свойство**: Данные легко передавать, сохранять и сравнивать.

## 2. Calculations (Вычисления)
Это чистые функции. Они принимают входные данные и возвращают результат.
- **Примеры**: Математические операции, форматирование строк, бизнес-логика (например, расчет скидки).
- **Свойство**: Они всегда возвращают один и тот же результат для тех же входов, их легко тестировать, они не зависят от времени запуска или внешнего мира.

## 3. Actions (Действия)
Это всё, что зависит от внешнего мира или меняет его (побочные эффекты).
- **Примеры**: Отправка Email, запрос к API, запись в базу, `console.log`, изменение глобальной переменной.
- **Свойство**: Действия непредсказуемы. Их результат может зависеть от того, сколько раз и когда они были вызваны.

## 4. Золотое правило архитектуры
1. **Выделяйте вычисления из действий**. Если у вас есть функция, которая скачивает данные и тут же их форматирует — разделите её на две.
2. **Делайте вычисления как можно больше, а действия — как можно меньше**.
3. **Действия должны находиться на "границах" системы** (например, в обработчиках кликов или в контроллерах API), а вся логика должна быть внутри чистых вычислений.

**Пример (плохо):**
```javascript
// Перемешано всё: и действие (запрос), и вычисление (налог)
async function showTotal() {
  const price = await fetchPrice();
  const total = price * 1.2; // Налог "зашит" внутри
  console.log(total);
}
```

**Пример (хорошо - ACD):**
```javascript
// Calculation (Чистое вычисление)
const calculateTax = (price) => price * 1.2;

// Action (Оркестрация действий и вычислений)
async function showTotal() {
  const price = await fetchPrice(); // Action
  const total = calculateTax(price); // Calculation
  console.log(total); // Action
}
```

## 5. Преимущества
- **Тестирование**: Вы можете протестировать 90% логики вашего приложения без моков (mocks), тестируя только чистые вычисления.
- **Повторное использование**: Вычисления не привязаны к источнику данных (API или ввод пользователя).
- **Надежность**: Сложные побочные эффекты локализованы в нескольких местах, а не размазаны по всему коду.
