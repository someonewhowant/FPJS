# Модуль 5: Замыкания в ФП (Closures)

Замыкание — это комбинация функции и лексического окружения, в котором эта функция была определена. Простыми словами, функция "запоминает" те переменные, которые были доступны ей в момент создания, даже если она вызывается вне своей первоначальной области видимости.

## 1. Как это работает?

Когда функция создается в JavaScript, она получает скрытую ссылку на область видимости, в которой она была создана.

**Пример (счетчик):**
```javascript
const createCounter = () => {
  let count = 0; // Эта переменная "захватывается" замыканием

  return () => {
    count++;
    return count;
  };
};

const counter = createCounter();

console.log(counter()); // 1
console.log(counter()); // 2
```
*Пошаговое объяснение:*
1. Функция `createCounter` вызывается и создает локальную переменную `count`.
2. Она возвращает анонимную функцию.
3. Даже после того, как `createCounter` завершила выполнение, возвращенная функция сохраняет доступ к переменной `count`.
4. Переменная `count` становится "приватной" — к ней нельзя обратиться напрямую, только через вызов функции `counter`.

## 2. Использование для конфигурации

Замыкания позволяют нам "предварительно настраивать" функции.

**Пример (генератор приветствий):**
```javascript
const makeGreeter = (greeting) => {
  return (name) => `${greeting}, ${name}!`;
};

const sayHello = makeGreeter('Hello');
const sayHi = makeGreeter('Hi');

console.log(sayHello('Alice')); // "Hello, Alice!"
console.log(sayHi('Bob'));      // "Hi, Bob!"
```

## 3. Почему это важно в ФП?
- **Инкапсуляция**: Мы можем скрывать состояние без использования классов и ключевого слова `this`.
- **Каррирование и Частичное применение**: Эти техники полностью полагаются на замыкания.
- **Мемоизация**: Мы можем хранить кэш результатов в замыкании, чтобы не пересчитывать их.

## 4. Память
Важно помнить, что переменные в замыкании не удаляются сборщиком мусора, пока жива сама функция, которая их использует. Это мощный инструмент, но его стоит использовать осознанно.
