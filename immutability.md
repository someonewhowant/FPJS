# Модуль 3: Иммутабельность (Immutability)

Иммутабельность (неизменяемость) — это концепция, согласно которой данные не могут быть изменены после их создания.

## 1. Почему это важно?
В императивном программировании мы часто меняем объекты "по месту" (in-place). В больших приложениях это приводит к тому, что трудно отследить, какая часть кода изменила состояние.

Иммутабельность дает:
- **Предсказуемость**: Вы точно знаете, что объект не изменится за вашей спиной.
- **История изменений**: Позволяет реализовать "Undo/Redo" или "Time Travel Debugging" (как в Redux).
- **Оптимизация**: Сравнение двух объектов сводится к сравнению ссылок (`objA === objB`), а не глубокому сравнению свойств.

## 2. Работа с массивами

### Избегайте мутирующих методов:
`push`, `pop`, `shift`, `unshift`, `splice`, `sort`, `reverse` — все они меняют исходный массив.

### Используйте функциональные аналоги:

**Добавление элемента:**
```javascript
const items = [1, 2, 3];
// Плохо: items.push(4);
const newItems = [...items, 4]; // [1, 2, 3, 4]
```

**Удаление элемента:**
```javascript
const items = [1, 2, 3];
const newItems = items.filter(item => item !== 2); // [1, 3]
```

**Обновление элемента:**
```javascript
const items = [1, 2, 3];
const newItems = items.map(item => item === 2 ? 20 : item); // [1, 20, 3]
```

## 3. Работа с объектами

### Используйте Spread-оператор (`...`)
```javascript
const user = { name: 'Alice', age: 25 };

// Обновление свойства
const updatedUser = { ...user, age: 26 };

// Добавление свойства
const userWithEmail = { ...user, email: 'alice@example.com' };
```

### Глубокое обновление
Важно помнить, что spread делает **поверхностную** копию.
```javascript
const company = {
  name: 'Tech',
  address: { city: 'NYC', street: 'Broadway' }
};

// Чтобы обновить город, нужно копировать все уровни вложенности:
const updatedCompany = {
  ...company,
  address: {
    ...company.address,
    city: 'San Francisco'
  }
};
```

## 4. Инструменты
Если вложенность объектов слишком глубокая, ручное копирование становится громоздким. В таких случаях используют библиотеки:
- **Immer**: Позволяет писать "мутирующий" код, который библиотека превращает в иммутабельный.
- **Immutable.js**: Предоставляет специальные структуры данных (List, Map).
