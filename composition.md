# Модуль 7: Композиция (Composition)

Композиция — это процесс объединения двух или более функций для создания новой функции. Это основной способ построения сложных программ в ФП из простых кирпичиков.

## 1. Проблема вложенных вызовов
Представьте, что нам нужно:
1. Очистить строку от пробелов.
2. Сделать её строчной.
3. Добавить восклицательный знак в конце.

**Обычный подход:**
```javascript
const trim = s => s.trim();
const lower = s => s.toLowerCase();
const exclaim = s => `${s}!`;

const result = exclaim(lower(trim('  HELLO  '))); // "hello!"
```
Читать такой код сложно: нужно начинать из центра (самого глубокого уровня) и двигаться наружу.

## 2. Pipe и Compose

Чтобы сделать код линейным и понятным, мы используем вспомогательные функции.

### Pipe (Конвейер)
Данные передаются слева направо (сверху вниз).
```javascript
const pipe = (...fns) => (x) => fns.reduce((acc, f) => f(acc), x);

const shout = pipe(
  trim,
  lower,
  exclaim
);

shout('  HELLO  '); // "hello!"
```

### Compose (Композиция)
Данные передаются справа налево (снизу вверх), как в математике `f(g(x))`.
```javascript
const compose = (...fns) => (x) => fns.reduceRight((acc, f) => f(acc), x);

const shoutCompose = compose(
  exclaim,
  lower,
  trim
);

shoutCompose('  HELLO  '); // "hello!"
```

## 3. Point-free стиль
Композиция позволяет нам писать код без явного указания аргументов (без "точек" данных).

**С аргументами:**
```javascript
const getNames = (users) => users.map(u => u.name);
```

**Point-free:**
```javascript
const map = f => list => list.map(f);
const getName = u => u.name;

const getNames = map(getName); // Мы не упоминаем 'users', мы просто создали функцию-трансформатор
```

## 4. Почему это круто?
- **Читаемость**: Код читается как список действий.
- **Модульность**: Каждую функцию можно тестировать отдельно.
- **Гибкость**: Легко добавить или убрать шаг в цепочке.
