# Модуль 9: Функциональные массивы

В ФП массивы обрабатываются не через циклы `for` или `forEach`, а через специальные методы трансформации.

## 1. Основная троица

### Map (Трансформация)
Создает новый массив, применяя функцию к каждому элементу.
```javascript
const numbers = [1, 2, 3];
const doubled = numbers.map(x => x * 2); // [2, 4, 6]
```

### Filter (Отбор)
Создает новый массив, оставляя только те элементы, для которых функция вернула `true`.
```javascript
const numbers = [1, 2, 3, 4];
const evens = numbers.filter(x => x % 2 === 0); // [2, 4]
```

### Reduce (Свертка)
Превращает массив в **одно** значение (число, строку, объект или даже другой массив).
```javascript
const numbers = [1, 2, 3, 4];
const sum = numbers.reduce((acc, x) => acc + x, 0); // 10
```

## 2. Почему Reduce — король?
`Reduce` настолько мощный, что через него можно реализовать любой другой метод массива.

**Пример (реализация map через reduce):**
```javascript
const myMap = (arr, fn) => 
  arr.reduce((acc, item) => [...acc, fn(item)], []);
```

## 3. Цепочки вызовов (Chaining)
Методы можно объединять в конвейер обработки данных.

**Пример (найти общую стоимость дорогих товаров):**
```javascript
const products = [
  { name: 'CPU', price: 300 },
  { name: 'Mouse', price: 20 },
  { name: 'GPU', price: 600 }
];

const totalExpensive = products
  .map(p => p.price)            // [300, 20, 600]
  .filter(price => price > 100) // [300, 600]
  .reduce((a, b) => a + b, 0);  // 900
```

## 4. Рекомендации
- Не используйте `forEach` в ФП, так как он предназначен для побочных эффектов.
- Всегда возвращайте значение из `map` и `reduce`.
- Не мутируйте массив внутри этих методов.
