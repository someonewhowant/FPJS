# Модуль 6: Каррирование (Currying)

Каррирование — это преобразование функции от многих аргументов в цепочку функций, каждая из которых принимает ровно один аргумент.

## 1. Отличие от обычного вызова

**Обычная функция:**
```javascript
const sum = (a, b, c) => a + b + c;
sum(1, 2, 3); // 6
```

**Каррированная функция:**
```javascript
const curriedSum = a => b => c => a + b + c;
curriedSum(1)(2)(3); // 6
```

## 2. Зачем это нужно?

Главное преимущество каррирования — возможность **частичного применения** (Partial Application). Мы можем передать часть аргументов и получить новую функцию, которая ожидает остальные.

**Пример (логирование):**
```javascript
const log = (importance) => (message) => {
  console.log(`[${importance}] ${message}`);
};

const info = log('INFO');
const error = log('ERROR');

info('Application started'); // [INFO] Application started
error('Crash detected');      // [ERROR] Crash detected
```
*Пошаговое объяснение:*
1. Мы создаем общую функцию `log`, которая принимает уровень важности.
2. Мы создаем специализированные функции `info` и `error`, зафиксировав первый аргумент.
3. Теперь нам не нужно каждый раз передавать строку `'INFO'`, мы вызываем уже готовую функцию.

## 3. Каррирование существующих функций

В библиотеках типа Lodash или Ramda есть функция `curry`, которая делает это автоматически.

**Пример (схематично):**
```javascript
const add = (a, b) => a + b;
const curriedAdd = _.curry(add);

const addTen = curriedAdd(10);
addTen(5); // 15
```

## 4. Каррирование vs Частичное применение
- **Каррирование**: Разбивает функцию на цепочку функций с *одним* аргументом.
- **Частичное применение**: Фиксирует *несколько* аргументов функции, возвращая новую функцию с меньшим количеством аргументов.

В JavaScript эти понятия часто используются взаимозаменяемо, так как каррирование — самый популярный способ добиться частичного применения.
